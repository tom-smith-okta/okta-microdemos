This demo shows a simple OIDC authentication flow using the Okta widget.
Log in as a user, and you will get an Okta session and an id token. The id token will include the user's group memberships.
The id token will be in local storage.
If you don't have a user yet, use the self-registration demo to create one.
This example flow is using the OAuth implicit flow for demo purposes.
In production, you should generally use the OAuth code flow (preferably with PKCE) whenever possible.
